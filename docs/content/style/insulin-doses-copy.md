# Гайд: терминология для болюсного и базального инсулина

Единый словарь помогает синхронизировать тексты бота, WebApp и справочных материалов.

## Термины в интерфейсах
| Контекст | Ключ | Значение для короткого | Значение для длинного | Пример использования |
|----------|------|------------------------|-----------------------|----------------------|
| Bot Reminders | `REMINDER_NAMES` | «Болюсный инсулин» | «Базальный инсулин» | "Напоминание: Болюсный инсулин в 08:30" |
| Bot Actions | `REMINDER_ACTIONS` | «Ввести болюсный инсулин» | «Ввести базальный инсулин» | Кнопки быстрого ответа |
| WebApp Forms | label | «Короткий (болюсный) инсулин» | «Длинный (базальный) инсулин» | Подписи к полям ввода |
| Push / Email | body | «болюс» | «базал» | «Вечерний базал: 10 ед.» |

## Общие правила
- При первом упоминании в тексте используйте пару «короткий (болюсный)» и «длинный (базальный)» — это помогает новичкам.
- В справке и подсказках явно указывайте, что значения задаются в единицах инсулина.
- Если речь про оба типа, используйте формулировку «короткий и длинный инсулин» вместо обобщённого «доза».
- Для нулевых значений пишите «не вводился», а не «0 ед.».

## Что НЕ писать
- Не смешивайте термины «короткий» и «быстрый» в одной фразе — выбирайте один стиль, предпочтительно «короткий (болюсный)».
- Избегайте слова «доза» без уточнения типа. Если тип неизвестен, поясните: «тип не указан (поле будет удалено)».
- Не используйте «инсулин обычный» / «инсулин длительный» — эти варианты не закреплены в глоссарии.
- Не просите пользователя копировать `dose` — поле устарело, новые интерфейсы должны работать только с `insulin_short` и `insulin_long`.
